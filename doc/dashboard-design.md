# Tick Dashboard UI 设计规范

**状态**: Draft  
**最后更新**: 2026-05-26  
**关联 PRD**: [tick-prd.md](./tick-prd.md) § 16.1  
**参考系统**: Cursor Dashboard（PC 版后台管理界面）

---

## 1. 设计目标

将 Dashboard 从当前"蓝色 SaaS 模板"风格改造为类 Cursor Dashboard 的极简克制风格。

**核心理念**: 克制、功能优先、编辑感、去装饰化、信息密度适中。

**一句话**: 不追求"好看"，追求"清晰"。颜色只在传递信息时出现，留白是主动设计手段。

---

## 2. 参考系统分析：Cursor Dashboard

### 2.1 整体布局

- 经典左侧边栏（~170px）+ 右侧内容区
- 内容区不设 max-width，自然居左，右侧大量留白
- 内容区内边距宽松（上 40-48px，左 48-64px）
- 区块间距 32-40px，用小号灰色标题做分区（非边框）
- 垂直节奏：各区块之间间距约 32-40px

### 2.2 设计哲学

> 整体像一本排版精良的技术杂志，不像一个 SaaS 模板。

- 颜色只在传递信息时出现（绿=数据、橙=注意、黑=行动）
- 装饰性元素几乎为零（无渐变、无阴影、无插图背景）
- 留白是主动设计手段，不是"没填满"
- 每屏聚焦一件事

## 3. 色彩系统

| 角色 | 色值 | Tailwind 近似 | 用法 |
|------|------|--------------|------|
| 侧边栏背景 | #F5F5F0 | stone-100 | 偏暖极浅灰 |
| 内容区背景 | #FAFAF8 / #FFFFFF | white | 主体底色 |
| 主标题 | #1a1a1a | gray-900 | 页面标题 |
| 正文 | #4a4a4a | gray-700 | 描述文字 |
| 次要文字 | #8a8a8a | gray-400 | 辅助说明、标签 |
| 导航选中背景 | #E8EDE8 | 自定义暖灰绿 | 浅背景 + 深色文字 |
| 点缀色-绿 | #10B981 | emerald-500 | 统计图表、状态指示、成功 |
| 点缀色-橙 | #F59E0B | amber-500 | 提示卡片、警告信息 |
| 主按钮 | #000000 | black | 黑底白字 |
| 次按钮 | 白底+灰框 | white + border | outline 样式 |
| 边框 | #E5E5E0 | stone-200 | 1px，偏暖灰，非常轻 |

**原则**: 整体无主色，黑白灰为主体。颜色仅作为信息载体（状态、警告、强调）出现。

## 4. 排版规范

### 4.1 字体栈

```
Inter, "Noto Sans SC", system-ui, -apple-system, sans-serif
```

Inter 通过 Google Fonts CDN 引入，中文回退到 Noto Sans SC。

### 4.2 字体层级

| 层级 | 元素 | 大小 | 粗细 | 颜色 | 用途 |
|------|------|------|------|------|------|
| H1 | 页面标题 | 24-28px | semibold (600) | #1a1a1a | 每页顶部唯一 |
| H2 | 区块标题 | 13-14px | regular (400) | gray-500 | 分区用，如"Source Control" |
| H3 | 卡片标题 | 15-16px | semibold (600) | #1a1a1a | 卡片或列表项标题 |
| Body | 正文 | 14px | regular (400) | #4a4a4a | 描述、说明 |
| Caption | 辅助文字 | 12-13px | regular (400) | gray-500 | 标签、时间戳 |
| Data | 数据大字 | 28-32px | semibold (600) | #1a1a1a | 统计数值 |

### 4.3 行高

- 标题：1.2 ~ 1.3
- 正文：1.5
- 辅助文字：1.4

## 5. 间距系统

| 场景 | 数值 | 说明 |
|------|------|------|
| 内容区上内边距 | 40-48px | 顶栏与正文之间 |
| 内容区左右内边距 | 48-64px | 内容与侧边栏/右边界 |
| 区块间距 | 32-40px | 两个独立内容区块之间 |
| 卡片内边距 | 20-24px | 卡片内部 |
| 列表行高度 | 48-56px | 带图标的列表行 |
| 导航项间距 | 4px | 侧边栏导航项之间 |
| 导航分组间距 | 16-20px | 导航分组之间（用空间代替分割线） |

## 6. 组件规范

### 6.1 侧边栏导航

- 宽度约 170-180px
- 背景色偏暖灰（stone-100）
- 分组用间距隔开（不用分割线）
- 选中态：浅灰绿圆角背景 + 深色文字，rounded-md
- 未选中：无背景，gray-600 文字
- 图标：16-18px Lucide outline 风格，与文字间距 8-10px
- 支持层级缩进（子项比父项缩进 12px）
- 底部固定：收起按钮 + 退出按钮

### 6.2 内容卡片

- 1px border（stone-200）+ rounded-lg
- 无 box-shadow
- 内部 padding 20-24px
- 列表行用细分割线隔开
- 每行：左侧图标+文字，右侧操作按钮
- hover：border 颜色轻微加深

### 6.3 统计卡片

- 横向等宽排列，1px 边框分格（连体格子，非独立卡片）
- 每格：上方小号灰色标签 + 下方大号黑色数字
- 可用绿色边框高亮重点指标

### 6.4 按钮体系

| 类型 | 样式 | 用途 |
|------|------|------|
| 主按钮 | 黑底白字，rounded-md，无 shadow | 核心操作：创建、连接、确认 |
| 次按钮 | 白底 + 1px 灰边框 + 灰色文字 | 次要操作：管理、取消 |
| 链接按钮 | 文字 + ↗ 图标，无底色无边框 | 跳转外部：Connect ↗ |
| 危险按钮 | 白底 + 红色文字 + 红色边框（hover） | 删除、撤销 |

### 6.5 提示/引导组件

- 引导清单：圆形 checkbox + 文字 + 右侧操作按钮
- 提示卡片：暖橙色浅背景（amber-50），居中布局，橙色文字标题
- 警告条：浅黄底 + 左侧三角图标 + 文字，全宽横幅
- Badge/Tag：浅灰底 + 小号文字，rounded-full

### 6.6 表单控件

- 输入框：1px border stone-200，rounded-md，focus 态 border 加深为 stone-400
- 标签：在输入框上方，13px gray-600
- 下拉/选择：与输入框同风格

## 7. 交互规范

| 交互 | 规范 |
|------|------|
| 链接 | 带下划线的深色文字（不用蓝色） |
| 外部链接 | 文字 + ↗ 箭头图标 |
| hover | 轻微背景色变化或 border 加深，无 scale/shadow |
| 过渡 | 150-200ms，ease-out |
| 进度指示 | 圆环/线性进度条，极简 |
| Toggle | 小号开关，灰色默认态 |

## 8. 与当前系统的改造对照

| 维度 | 当前 | 目标 | 改造动作 |
|------|------|------|----------|
| 色温 | 冷灰 gray-* | 暖灰 stone-* | 全局色系替换 |
| 主色 | blue-500/600 | 无主色，黑灰+微量绿 | 去蓝化 |
| 导航高亮 | bg-blue-50 + text-blue-600 | 浅灰绿背景 + 深色文字 | 柔和选中态 |
| 按钮 | 蓝色系 | 黑底白字/白底灰框 | 按钮系统重做 |
| 间距 | p-6（24px） | p-8~p-12（32-48px） | 全面加大留白 |
| 顶栏 | 带边框 header bar | 极简或去掉 | 简化 |
| 字体层级 | 单一大小 | 5 个明确层级 | 建立字体阶梯 |
| 卡片 | 无明显卡片风格 | 1px border 无 shadow | 统一容器风格 |
| 装饰 | 蓝色图标点缀 | 零装饰 | 去除所有装饰色 |
| 字体 | system-ui 默认 | Inter + Noto Sans SC | 引入设计字体 |

## 9. 改造范围

### 包含

- Layout 组件（侧边栏 + 顶栏 + 内容区）
- 所有页面：任务列表、任务详情、任务创建/编辑、API Key、配额、设置
- 登录/注册页
- 全局样式（index.css / tailwind config）

### 不包含

- 后端 API 变动
- 功能逻辑变更
- 深色模式（未来扩展，本次不做）
- 移动端专属适配（保持现有响应式，仅调整色彩间距）
